Introduction to the Web

The world wide web is a collection of HTML documents. HTML is the basis for almost every web page. HTML is what glues everything together. The links between these pages are called hyperlinks. The components of the web are:

  • Browser - A program that runs on your computer to display files found on the web.
  • Internet - The world's largest computer network.
  • HTTP - The main protocol of the web.
  • Servers - Computers that host the files that make up the web.

HTML Basics

HTML stands for Hypertext Markup Language. HTML documents are the heart of the web and are made up of:

  • Text Content - what you see
  • Markup - what it looks like
  • References to Other Documents - e.g. images & videos
  • Links to Other Pages

Scratchpad is a great, easy to use real-time HTML & CSS editor.

Intro to HTML Tags

HTML markup is made up of HTML elements. Elements types are defined by tags. Tags generally look like this:

<tag>content</tag>

Contents in the element are surrounded by an opening tag and a closing tag. The opening and closing tags have the same name, except the closing tag has a slash (/) before the name to indicate it is a closing tag.

Computers Are Stupid

An important point about how computers understand instructions is that computers cannot make inferences based on context. Computers interpret language literally so it is important to keep in mind how sensitive computers can be to typo and small human mistakes. These small errors can lead to huge problems when given to a 'stupid' computer.

Adding Images

Using the image tag, you can reference an image on your website. An image tag looks like this:

<img src="url" alt="text">

The attribute called 'alt' (alternate) is text that gets displayed in case the image doesn't load. Forgetting the 'alt' attribute will not break the page, but it is easy and courteous to include.

It's OK to Forget

It might seem like you need to memorize every single tag, attribute, and property. However programming has too many facts to remember them all. You only need to have a solid understanding of a few big ideas and know how to get the details when you need them. There are plenty of resources online to help you out such as Google and W3Schools Online Web Tutorials.

Inline vs Block

HTML elements are either inline or block. These elements have different behavior.

An inline element occupies only the space of the content in the element. Some examples of inline elements are:

  • <b>
  • <em>
  • <a>
  • <span>
  • <br>
  • <img>

Block elements actually create an invisible box around the content inside of them. This invisible box can have its own properties like height and width. Some examples of block elements are:

  • <p>
  • <div>
  • <h1>
  • <form>

Boxes and Containers?

Container elements are elements that do nothing but contain things inside themselves. These elements will be covered in the next lesson.

Document Structure

The concept of code structure is a very important idea in computer programming. A huge part of well written code is structure. Code should be written so that people can read it easily.

Google Chrome Development Tools

Otherwise known as "Dev Tools", Google Chrome Development Tools lets you take a look at the source code of a web page to better understand what is going on. It allows you to inspect and change the original code, but any changes are not saved. You can simply refresh the browser to reset to the original code. This is an extremely powerful learning tool as it allows you to view and analyze each element on the web page.

Branching Tree-Like Structures

When inspecting a page using Dev Tools, you can see that several elements are grouped together (such as the <body> tag). Sideways triangles mean there is a grouped HTML element. Upon expanding these groups, each element has an opening tag and a closing tag. These elements can enclose text, or another element (such as the <h1> or <div> tags). You can keep expanding deeper into these tags, much like the branches of a tree where each branch can split into additional smaller branches.

Indentation and Nesting Elements

When writing HTML, it is important to use proper indentation. Much like how Chrome Dev Tools groups nested elements/branches with a triangle icon, you should always indent your HTML elements for readability. This makes the code easier to read and understand.

Div, Span, ID, and Class

Div, span, ID, and class are tags used to provide structure to HTML and style using CSS. These tags are used as containers, holding other tags or content inside. Once structure in HTML, you can use CSS to style the tags. This is a great tool in eliminating repetition and writing good code.

HTML Validation

A code validator is an online program that allows you to check your HTML code for syntax errors. Code validation cannot ensure that your web page appears the way you want it- but it can prevent syntax errors. Even if you don't understand the error massage, a validator can show you exactly which lines in the code you need to double check. You can input you code directly for HTML validation.

CSS Styling

CSS stands for Cascading Style Sheets and allows you to give your page style (colors, shapes, fonts, etc.) You can apply CSS elements to entire HTML classes. You can embed your CSS inside the <head> tags of a document, or you can attach a separate file where you define the styles externally.

Why use CSS styling?

CSS allows you to define styles in one place that you can reference more than once. This is a powerful and efficient tool to eliminate repetition. Using one style sheet also helps make maintenance and site-wide updates much easier.

CSS Validation

It is also helpful to validate CSS to avoid syntax errors.
